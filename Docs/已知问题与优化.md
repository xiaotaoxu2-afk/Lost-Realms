## 1. 前言
本项目核心玩法已完成，实现完整游戏流程，但存在部分不影响核心体验的小bug。本文档记录已知问题、原因分析，并制定优化计划，体现持续改进意识。


## 2. 已知小bug（不影响核心游戏体验）
| bug现象 | 产生原因分析 | 暂未修复说明 |
|--------|--------------|--------------|
| 部分攻击音效未触发（概率约5%） | 音频资源Addressables异步加载延迟，未添加播放失败的降级处理 | 仅轻微影响体验，后续通过调整加载顺序修复 |
| 背景图片显示错误 | Addressables资源依赖配置遗漏，导致资源加载异常 | 无实际体验影响，后续统一检查资源依赖 |


## 3. 潜在问题（未实际复现，存在理论风险）
| 潜在问题 | 风险分析 | 预防方案 |
|----------|----------|----------|
| 长时间游戏后（>30分钟）出现内存泄漏 | Addressables资源释放逻辑存在遗漏，长期运行导致内存累积 | 使用Unity Memory Profiler进行长时间运行测试，定位泄漏点并完善释放逻辑 |
| 低配置PC（如Intel核显）出现帧率波动 | 未针对低端设备做专项性能优化 | 新增“画质设置”选项，低画质模式下关闭非核心特效、降低DrawCall |


## 4. 后续优化计划（分阶段实施）
### 4.1 第一阶段：修复小bug（1-2周）
| 优化内容 | 具体实施方法 | 预期效果 |
|----------|--------------|----------|
| UI自适应优化 | 给UI面板添加`Layout Group`+`Content Size Fitter`组件，重新调整锚点 | 支持1080P/720P/4K等分辨率，切换窗口无UI错位 |
| 音效播放稳定性优化 | 1. 调整音频资源的Addressables加载优先级；2. 在`AudioTool.cs`中添加播放失败的降级处理（重新加载+日志提示） | 攻击音效触发成功率提升至99%以上 |


### 4.2 第二阶段：性能与体验优化（2-3周）
| 优化内容 | 具体实施方法 | 预期效果 |
|----------|--------------|----------|
| 内存泄漏修复 | 使用Unity Memory Profiler进行30分钟以上运行测试，定位未释放资源并完善`Addressables.Release`逻辑 | 长时间游戏后内存占用稳定，无明显累积 |
| 低配置设备适配 | 新增“画质设置”面板，低画质模式下：关闭粒子特效、合并Sprite图集、降低动画帧率 | 低配置PC帧率稳定在60FPS |
| 战斗打击感提升 | 1. 给角色攻击添加轻微屏幕震动；2. 敌人受伤时添加伤害飘字；3. 同步Boss技能的特效与音效 | 战斗反馈更清晰，提升体验沉浸感 |


### 4.3 第三阶段：功能扩展（3-4周）
| 扩展内容 | 具体实施方法 | 预期效果 |
|----------|--------------|----------|
| 新手引导系统 | 在第一关卡添加UI箭头+文字提示，引导移动/攻击/跳跃等核心操作 | 降低新手玩家学习成本，提升留存率 |
| 多语言支持 | 通过ScriptableObject存储多语言文本（中文/英文），实现语言切换功能 | 适配海外玩家，提升项目扩展性 |
